About This Program

GreenGate values the security community and believes that responsible disclosure of vulnerabilities makes our services safer for everyone. This program defines how you can help us identify and fix security issues.

Vulnerability Types

Critical Priority CRITICAL

  • Remote code execution (RCE)
  • SQL injection with access to sensitive data
  • Mass data leak (API keys, customer data)
  • Complete authentication bypass
  • Report code enumeration (successful brute-force)

High Priority HIGH

  • Cross-Site Scripting (XSS) allowing API key theft
  • Cross-Site Request Forgery (CSRF) on critical operations
  • Personal data exposure (CPF, CNPJ, names)
  • Authorization bypass (access to other customers' reports)
  • Rate limiting bypass enabling enumeration

Medium Priority MEDIUM

  • Reflected XSS without access to sensitive data
  • Technical information disclosure (versions, paths)
  • Clickjacking on non-critical pages
  • Missing security headers

Low Priority LOW

  • Misconfigured SPF/DKIM/DMARC
  • Cookies without Secure flag (in non-sensitive contexts)
  • UI/UX issues that don't impact security
‚ö†Ô∏è Not Vulnerabilities:
  • Absence of best practices without real impact (e.g., missing HSTS if HTTPS already redirects)
  • Automated scanner reports without manual validation
  • Theoretical vulnerabilities without proof-of-concept
  • Issues in third-party services (Cloudflare, Railway, etc.)

How to Report

Contact Information

Email: greengatebrasil@gmail.com

Subject: [SECURITY] Brief vulnerability description

PGP Key: Download public key (optional but recommended)

What to Include in Your Report

  1. Clear description: Summary of the vulnerability and potential impact
  2. Steps to reproduce: Detailed step-by-step instructions
  3. Proof of concept: Screenshots, videos, or demonstrative code
  4. Environment: Browser, OS, tools used
  5. Impact: Your severity assessment (Critical/High/Medium/Low)
  6. Fix suggestions: If you have any (optional)

Rules of Conduct

‚úÖ Allowed

‚ùå Prohibited

Disclosure Process

  1. Private report: You send vulnerability to greengatebrasil@gmail.com
  2. Confirmation: We confirm receipt within 2 business days
  3. Triage: We assess severity and impact within 5 business days
  4. Fix: We develop and deploy fix (30-90 days based on severity)
  5. Validation: We ask you to validate the fix
  6. Recognition: We add you to Hall of Fame (if desired)
  7. Coordinated disclosure: After 7 days post-fix, we may disclose publicly (with your consent)

Frequently Asked Questions

Can I test in production?

Yes, but with moderation. Avoid destructive actions, use test data, and don't impact other users.

What if I accidentally cause damage?

Report immediately to greengatebrasil@gmail.com. If you follow this policy in good faith, we won't take legal action.

Can I use automated scanners?

Yes, but validate manually before reporting. Don't send automated reports without analysis.

Do you offer a test environment?

We currently don't have public staging. Use test data in production with caution.

Do you offer financial rewards?

No. This is a responsible disclosure program focused on public recognition (Hall of Fame). We don't offer monetary bounties at this time.
